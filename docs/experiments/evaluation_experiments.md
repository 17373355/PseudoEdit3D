# Evaluation Experiments

## Evaluation Goal

The current evaluation is split into three conceptual layers:

1. annotation-layer evaluation
2. generation-probe evaluation
3. future structured-condition model evaluation

The current work mainly focuses on the first two.

## 实验 1：HumanML3D 重标注基准

在相同的 MoMask 架构下进行比较。这里保持相同的 tokenizer / decoder family，但使用不同的 text-conditioned transformer 监督信号：

- 原始 HumanML3D captions + 使用原始 HumanML3D 训练的 MoMask text-conditioned model -> FID
- AutoPrompt-HumanML3D + 使用 AutoPrompt-HumanML3D 重新训练的 MoMask text-conditioned model -> FID

### 该实验可以支持什么结论

- annotation layer 会显著影响 text-to-motion 生成质量。
- AutoPrompt 提供了一个更一致、噪声更低的 conditioning space。
- 可以通过替换原始 textual supervision，并在相同 MoMask 架构下重新训练 text-conditioned transformer，构建一个新的 AutoPrompt-HumanML3D benchmark。

### 该实验单独不能证明什么

- 每一条 caption 的语义都完全正确。
- AutoPrompt 已经达到完整的 atomic granularity。
- 数量、角度、幅度等细粒度属性已经被精确 grounding。

## 实验 2：跨数据集泛化

在无标签数据集 B 上：

- motion_B -> AutoPrompt
- 使用 AutoPrompt-HumanML3D 训练得到的 AutoPrompt-conditioned model -> generation
- generated motion vs GT_B -> FID

### 该实验可以支持什么结论

- AutoPrompt 并不只是拟合 HumanML3D 的文本表述风格。
- 学到的 condition space 具有更好的跨数据集迁移能力。
- AutoPrompt 可以作为一种可复用的 motion semantic interface，用于无标签 motion data 的自动语义标注和条件生成。

### 该实验单独不能证明什么

- AutoPrompt 对所有 motion distributions 都具有普适泛化能力。
- AutoPrompt 在所有 atomic categories 上都具有同等质量。

## Current Evaluation Split

### Annotation-layer evaluation

- `good`
- `soft_bad`
- `hard_bad`

### Generation-probe evaluation

- `selected_hml3d_prompt -> MoMask`
- `auto_prompt -> MoMask`

## Important Constraint

For correctness studies, `auto_prompt` must be motion-only.
It must not be generated by retrieving same-case HumanML3D captions.

## Current Priority Failure Classes

- false crouch / false squat
- walk_backward missing
- stop_pause edge cases
- repeated hop vs repeated squat vs repeated bounce
- limb-level repeated arm patterns
- hand support / rail support

### Recommended Supporting Analyses for Experiment 1

To make the argument stronger, Experiment 1 should be accompanied by the following analyses.

#### A. Annotation-Layer Consistency Analysis

Compare directly on HumanML3D:

- conflict rate among multiple original captions for the same motion
- normalization level of AutoPrompt
- prompt consistency among similar motions
- lower expression entropy for the same semantic pattern

These analyses directly support the claims that AutoPrompt is:

- more uniform
- less noisy
- more standardized

#### B. Prompt-to-Motion Semantic Consistency Cases

Use qualitative visualization with:

- selected HML3D prompt
- auto prompt
- generated motion
- GT motion

These cases support claims such as:

- more accurate semantic capture
- more fine-grained atomic description

### Recommended Controls and Extensions for Experiment 2

#### A. Zero-shot vs Benchmark-trained Comparison

If possible, compare on dataset B:

- original HML3D-caption-trained MoMask
- AutoPrompt-HML3D-trained MoMask

This makes it easier to argue that:

- not every HumanML3D-trained model transfers equally well
- AutoPrompt-based supervision improves transferability

#### B. Round-trip Consistency

A useful consistency test is:

```text
motion_B
-> AutoPrompt
-> MoMask generation
-> generated motion
-> re-extract AutoPrompt
```

Then compare:

- original AutoPrompt
- re-extracted AutoPrompt

If consistency is high, that supports the claim that AutoPrompt is a stable motion semantic language rather than a superficial text layer.

## Recommended Claim Wording

### Experiment 1 Main Claim

#### English

Under the same MoMask architecture, replacing the original HumanML3D captions with our AutoPrompt-HumanML3D supervision and re-training the text-conditioned transformer leads to better FID. This supports the view that AutoPrompt forms a more consistent, lower-noise, and more learnable conditioning space than the original HumanML3D captions.

#### Chinese

在保持 MoMask 架构不变的前提下，将 HumanML3D 原始文本标注替换为我们构建的 AutoPrompt-HumanML3D 标注，并重新训练 text-conditioned transformer，可获得更优的 FID。这说明 AutoPrompt 构成了一个比原始 HumanML3D 标注更一致、噪声更低、也更利于模型学习的条件语义空间。

### Experiment 1 Stronger Benchmark Framing

#### English

This experiment also supports treating AutoPrompt-HumanML3D as a new re-annotated benchmark, where the gain comes from improved supervision quality rather than from changing the generative backbone.

#### Chinese

该实验还支持将 AutoPrompt-HumanML3D 视为一个新的重标注基准：性能提升主要来自监督信号质量的改善，而不是生成骨干网络的改变。

### Experiment 2 Main Claim

#### English

If a model trained on AutoPrompt-HumanML3D can generalize to an unlabeled dataset B by conditioning on automatically extracted AutoPrompts and still achieve low FID against the target motions, this supports the view that AutoPrompt acts as a transferable motion semantic interface rather than a benchmark-specific caption style.

#### Chinese

如果一个在 AutoPrompt-HumanML3D 上训练得到的模型，能够在无标签数据集 B 上仅依赖自动提取的 AutoPrompt 进行条件生成，并仍然取得较低的 FID，那么这支持如下观点：AutoPrompt 并不是一种仅适配 HumanML3D 的 benchmark 文本风格，而是一种具有跨数据集迁移能力的 motion semantic interface。

### Experiment 2 More Careful Interpretation

#### English

This does not by itself prove universal generalization across all motion domains, but it does support that the learned conditioning space captures transferable motion semantics beyond the original benchmark language.

#### Chinese

这并不能单独证明它对所有运动域都具有普适泛化性，但它足以支持这样一个结论：所学习到的条件空间捕捉到了超越原 benchmark 文本风格的、可迁移的 motion semantics。
